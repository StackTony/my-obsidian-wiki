# B树与B+树

树结构平衡了数组查询O(1)和链表插入删除O(n)的差距。B树和B+树是多路搜索树，专门优化磁盘IO场景。

## 二叉查找树 → AVL → 红黑树 → B树的演进路线

| 树 | 平衡方式 | 适合场景 |
|----|----------|----------|
| BST | 不平衡 | 随机数据插入 |
| AVL | 严格平衡(高度差≤1) | 查找远多于增删 |
| 红黑树 | 弱平衡(最长≤2×最短) | 增删查频率相近 |
| B/B+树 | 多路矮胖 | 磁盘IO/数据库索引 |

## B树（B-Tree）

m阶B树的关键约束：

1. 非叶子节点最多m个子节点（m>2）
2. 根节点子节点数∈[2, m]
3. 非根非叶子节点子节点数∈[⌈m/2⌉, m]
4. 每个节点存放⌈m/2⌉-1 到 m-1 个关键字
5. 非叶子节点关键字数 = 子节点指针数 - 1
6. 关键字有序：K[1] < K[2] < ... < K[m-1]
7. 指针P[i]指向关键字范围(K[i-1], K[i])的子树
8. 所有叶子节点位于同一层

**搜索**：从根节点二分查找，命中则结束，否则进入对应范围的子节点，重复直到叶子节点或指针为空。

特性：关键字分布在整棵树中、搜索可能在非叶子节点结束、搜索性能等价于全集二分查找。

## B+树

B+树是B树的变体，关键差异：

| 差异点 | B树 | B+树 |
|--------|-----|------|
| **关键字数量** | m个子节点，m-1个关键字 | m个子节点，m个关键字 |
| **数据位置** | 每个节点都存数据 | **所有数据只在叶子节点** |
| **分支节点** | 存关键字+数据+子指针 | 仅存关键字+子指针（纯索引） |
| **叶子链表** | 无 | **叶子节点有链指针** |
| **搜索** | 可在非叶子节点命中 | 必须走到叶子节点才命中 |

## 为什么B+树更适合数据库索引

1. **磁盘IO代价更低** — B+树内部节点仅存关键字（不含数据指针），同盘块容纳更多关键字 → 一次IO读入更多关键字 → IO次数更少

2. **查询效率更稳定** — 任何查找都走根→叶子的完整路径，路径长度相同，查询时间稳定

3. **范围查询高效** — 叶子节点链表支持顺序遍历，B树范围查询效率低（需要中序遍历整棵树）

这些特性使得B+树成为数据库索引和文件系统的首选结构。^[inferred]
