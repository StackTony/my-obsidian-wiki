## 四、学习路径建议

### 入门路径（理解概念）

```
可观测性三支柱 → Prometheus 基础 → Grafana 可视化 → ELK 日志 → 分布式追踪
     ↓              ↓              ↓           ↓            ↓
  理解区别      Pull模型/TSDB    Dashboard   ES/Logstash    Trace/Span
```

### 进阶路径（架构设计）

```
OpenTelemetry 统一标准 → SkyWalking APM → 生产级架构演进
        ↓                    ↓                ↓
  Traces/Metrics/Logs    Java Agent       Kafka+ES 集群
```

### 推荐阅读顺序

1. **第一周**：Prometheus + Grafana（指标监控）
   - 图解 Prometheus 原理 → Prometheus 架构详解 → Grafana 深度解析
2. **第二周**：ELK Stack（日志分析）
   - ELK 原理和实操 → ELK 架构演进 → Filebeat 替代方案
3. **第三周**：OpenTelemetry + SkyWalking（分布式追踪）
   - OpenTelemetry 基础知识 → SkyWalking 原理分析 → 生产级案例

## 六、监控体系核心术语表

### 可观测性基础

| 术语 | 定义 |
|------|------|
| **Observability（可观测性）** | 通过外部输出推断系统内部状态的能力 |
| **Metrics（指标）** | 定量测量数据（CPU/内存/QPS/延迟） |
| **Logs（日志）** | 带时间戳的事件记录 |
| **Traces（链路）** | 分布式系统中请求的完整执行路径 |
| **SLI（Service Level Indicator）** | 服务质量指标（如延迟、错误率） |
| **SLO（Service Level Objective）** | 服务质量目标（如"99.9%请求<100ms"） |
| **SLA（Service Level Agreement）** | 服务质量协议（合同承诺） |

### Prometheus 核心术语

| 术语 | 定义 |
|------|------|
| **TSDB（时序数据库）** | 存储带时间戳的数值序列 |
| **Pull Model** | 主动拉取指标，被监控端暴露 HTTP endpoint |
| **Exporter** | 转换第三方系统指标为 Prometheus 格式的代理 |
| **Pushgateway** | 接收短期任务推送指标的中间层 |
| **PromQL** | Prometheus 查询语言，支持时序数据聚合 |
| **Alertmanager** | 告警管理组件，支持分组/静默/路由 |
| **四种指标类型** | Counter（累加）、Gauge（瞬时）、Histogram（分布）、Summary（分位数） |

### Grafana 核心术语

| 术语 | 定义 |
|------|------|
| **Dashboard** | 可视化仪表板，包含多个 Panel |
| **Panel** | 单个可视化组件（图表/表格/地图） |
| **Data Source** | 数据源连接（Prometheus/ES/InfluxDB 等） |
| **Annotation** | 事件标注，在图表上标记重要事件 |
| **Alert Rule** | 告警规则，基于查询条件触发通知 |

### ELK Stack 核心术语

| 术语 | 定义 |
|------|------|
| **Elasticsearch** | 分布式全文搜索引擎，存储日志索引 |
| **Logstash** | 数据收集处理引擎（Input→Filter→Output） |
| **Kibana** | 可视化分析平台，查询和展示 ES 数据 |
| **Beats** | 轻量级数据采集器（Filebeat/Packetbeat/Winlogbeat） |
| **Index** | ES 中数据的逻辑容器（类似数据库表） |
| **Document** | ES 中最小数据单元（JSON 格式） |

### OpenTelemetry 核心术语

| 术语 | 定义 |
|------|------|
| **OTel（OpenTelemetry）** | CNCF 可观测性统一标准框架 |
| **Span** | 链路中的基本构建块，代表单个操作 |
| **Trace** | 由多个 Span 组成的请求执行路径（DAG） |
| **SpanContext** | TraceID + SpanID + Baggage，跨进程传递 |
| **Baggage** | Span 的随行数据（键值对），全链路传递 |
| **Collector** | 遥测数据收集器，支持处理和导出 |
| **Instrumentation** | 代码埋点，生成遥测数据 |

### SkyWalking 核心术语

| 术语 | 定义 |
|------|------|
| **APM（Application Performance Monitoring）** | 应用性能监控系统 |
| **OAP（Observability Analysis Platform）** | SkyWalking 服务端分析平台 |
| **Agent** | 字节码增强探针，无侵入收集数据 |
| **Service** | 被监控的应用服务实例 |
| **Endpoint** | 服务暴露的 API 接口 |
| **Topology** | 服务调用拓扑图 |
| **Segment** | 单次请求的完整调用链数据 |
