## 九、消息队列核心术语表

### Kafka 核心术语

| 术语 | 定义 |
|------|------|
| **Broker** | Kafka 集群中的服务器节点，负责消息存储和转发 |
| **Topic** | 消息的逻辑分类，Producer 发送、Consumer 订阅的主题 |
| **Partition** | Topic 的物理分段，分布在不同 Broker 上实现并行处理 |
| **Replica** | Partition 的副本，保障高可用（Leader + Follower） |
| **ISR（In-Sync Replicas）** | 与 Leader 保持同步的副本集合，AR=ISR+OSR |
| **HW（High Watermark）** | 高水位，消费者最多能消费到的 offset 位置 |
| **LEO（Log End Offset）** | 下一条待写入消息的 offset |
| **Consumer Group** | 消费者组，同组内每条消息只被一个 Consumer 消费 |
| **Offset** | 消息在 Partition 中的位置索引，消费进度标记 |
| **Segment** | Partition 的物理存储分段（.log + .index 文件） |
| **acks** | 生产者确认级别：0=不等确认、1=Leader确认、-1=ISR全部确认 |

### RabbitMQ 核心术语

| 术语 | 定义 |
|------|------|
| **Exchange** | 消息路由代理，接收 Producer 消息并按规则分发到 Queue |
| **Direct Exchange** | 精确匹配 Routing Key，适用于单播路由 |
| **Fanout Exchange** | 广播到所有绑定 Queue，忽略 Routing Key |
| **Topic Exchange** | 通配符匹配 Routing Key（* = 一个词，# = 多个词），适用于多播 |
| **Headers Exchange** | 基于 Headers 属性键值对匹配，替代 Routing Key |
| **Queue** | 消息存储载体，FIFO 管理，等待 Consumer 消费 |
| **Binding** | Exchange 与 Queue 的绑定关系，定义路由规则 |
| **Routing Key** | 路由键，Producer 发送消息时指定，Exchange 据此路由 |
| **VHost** | 虚拟主机，资源隔离单元，不同 VHost 的 Exchange/Queue 完全隔离 |
| **Channel** | 连接内的轻量级会话通道，减少 TCP 连接开销 |
| **Dead Letter Queue** | 死信队列，无法消费的消息重入队或转存 |

### 消息队列通用术语

| 术语 | 定义 |
|------|------|
| **异步（Async）** | 生产者不等待消费者处理结果，立即返回 |
| **解耦（Decoupling）** | 生产者和消费者通过 MQ 中间层独立演进 |
| **削峰（Peak Shaving）** | MQ 缓冲高峰流量，消费者按自身速率处理 |
| **at-most-once** | 消息最多被消费一次（可能丢失） |
| **at-least-once** | 消息至少被消费一次（可能重复） |
| **exactly-once** | 消息恰好被消费一次（最理想但最难实现） |
