## 九、向量数据库核心术语表

### 向量数据库核心术语

| 术语 | 定义 |
|------|------|
| **Embedding** | 将非结构化数据（文本/图像/音频）转换为高维数值向量的过程，由模型（BERT/ResNet）生成 |
| **ANN（Approximate Nearest Neighbor）** | 近似最近邻搜索，在精度与速度间权衡，避免全量遍历 |
| **KNN（K-Nearest Neighbor）** | 精确最近邻搜索，100% 召回率但计算量随数据量线性增长 |
| **Collection** | Milvus 中数据存储的顶层容器，需预定义 schema（向量字段+标量字段） |
| **Segment** | Collection 的物理存储分段，Milvus 数据管理的基本单元 |
| **Partition** | Collection 的逻辑分区，基于标量字段值划分，加速查询过滤 |

### 索引类型术语

| 术语 | 定义 |
|------|------|
| **HNSW（Hierarchical Navigable Small World）** | 分层导航小世界图索引，上层稀疏快速导航、下层稠密精准匹配，O(log N) 查询复杂度 |
| **IVF_FLAT** | 倒排文件索引，K-Means 聚类分桶后桶内存储原始向量，召回率最优但内存占用最高 |
| **IVF_SQ8** | IVF + 标量量化，float32→uint8 压缩为 1/4，内存降低 70-75%，精度轻微损失 |
| **IVF_PQ** | IVF + 乘积量化，向量分段独立量化编码，97% 内存压缩但精度损失较大 |
| **DiskANN** | 磁盘优化索引，适用于超大规模（>1 亿）低内存成本场景，查询延迟较高 |
| **FLAT** | 暴力检索索引，全量遍历无压缩，100% 召回率，适合万级以下小规模数据 |

### HNSW 核心参数

| 参数 | 定义 | 推荐范围 |
|------|------|----------|
| **M** | 每个节点的最大连接数（连接度），越大图越精细但内存和构建时间增加 | 5-100 |
| **efConstruction** | 构建时候选最近邻居数，越大索引质量越高但构建时间越长 | 100-2000 |
| **ef（efSearch）** | 搜索时候选最近邻居数，越大召回率越高但查询时间越久 | top_k~2000 |

### IVF 核心参数

| 参数 | 定义 | 推荐范围 |
|------|------|----------|
| **nlist** | 聚类分桶数量，越多分类越精细但构建越慢 | [1, 65536]，默认 128 |
| **nprobe** | 搜索时访问的聚类桶数，越多召回率越高但延迟越大 | [1, nlist]，默认 8 |

### 距离度量术语

| 术语 | 定义 |
|------|------|
| **Cosine（余弦相似度）** | 衡量向量方向相似性，归一化后范围 [0, 1]，适合文本语义搜索 |
| **L2/Euclidean（欧氏距离）** | 衡量向量空间绝对距离，适合图像特征匹配 |
| **IP（Inner Product/内积）** | 向量点积，适合已归一化向量的相似度计算 |
| **Hamming（汉明距离）** | 二值向量间不同位的数量，仅适用于二值向量 |

### 量化压缩术语

| 术语 | 定义 |
|------|------|
| **PQ（Product Quantization/乘积量化）** | 将高维向量分割为子向量，各子向量独立聚类编码，97% 内存压缩 |
| **SQ（Scalar Quantization/标量量化）** | float32→int8 离散化映射，4x 压缩，精度损失小 |
| **BQ（Binary Quantization/二进制量化）** | float32→1bit 二值表示，32x 压缩，精度损失最大 |

### Milvus 架构术语

| 术语 | 定义 |
|------|------|
| **Access Layer（访问层）** | 接收客户端请求，路由至对应组件 |
| **Coordinator Service（协调服务）** | 管理集群拓扑、负载均衡、元数据协调 |
| **Worker Node（工作节点）** | Data Node（写入）/ Index Node（索引构建）/ Query Node（查询） |
| **Storage Layer（存储层）** | 元数据存储（etcd）+ 消息存储（Pulsar/Kafka）+ 对象存储（MinIO/S3） |
| **存算分离** | 计算节点与存储节点独立扩展，云原生核心设计 |
