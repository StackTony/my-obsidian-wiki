## 四、核心知识点整理

### 1. Chunk策略图谱

```
Level 1: Character Splitting
    └── 固定500字符切分，无语义感知（生产不推荐）

Level 2: Recursive Character Splitting ⭐ 生产默认
    └── Separator层级: ["\n\n", "\n", ". ", " ", ""]
    └── 配置: 256-512 token + 10-15% overlap
    └── Chroma基准: 400-512 token + 10-20% overlap → 85-90% Recall@5

Level 3: Document-Structure Splitting
    └── Markdown/HTML/LaTeX结构感知
    └── 变长分块，保持主题完整

Level 4: Semantic Chunking
    └── 嵌入相似度检测话题转变
    └── 基准召回91-92%，端到端仅54%（分块太小）
    └── ⚠️ 必须200 token最小限制

Level 5: Agentic Chunking
    └── LLM判断分块边界
    └── 最高质量，100x成本

Level 6: Parent-Child Chunking
    └── 小子分块(100-500 token)检索
    └── 大父分块(500-2000 token)生成
    └── 解决精度vs上下文权衡
```

### 2. Chunk反模式识别

| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 重叠>20% | 精度下降、索引膨胀2-3倍 | 10-20%重叠是黄金分割 |
| 单策略处理异构语料 | 专利/chat/API文档效果都差 | 按文档类型分块逻辑 |
| 语义分块无最小限制 | 43 token碎片LLM无法使用 | 强制200+ token最小 |
| 不验证分块边界 | 句中切分、表格标题分离 | 手动抽查50分块 |

### 3. Rerank二阶段检索

```
第一阶段：粗排（召回）
┌─────────────────────────────────────┐
│  Bi-Encoder (Embedding模型)         │
│  Query独立编码 → 向量               │
│  Doc独立编码 → 向量                 │
│  → 向量相似度排序                   │
│  优势: 快、可ANN加速、可预索引      │
│  精度: 70-80%                       │
└─────────────────────────────────────┘
              ↓ Top 50-100候选

第二阶段：精排（Rerank）
┌─────────────────────────────────────┐
│  Cross-Encoder (Reranker)           │
│  Query + Doc拼接输入                │
│  → Transformer联合理解              │
│  → 相关性评分(0-1)                  │
│  优势: 精度高、上下文理解强         │
│  精度: 95%+                         │
│  劣势: 慢、无法预索引               │
└─────────────────────────────────────┘
              ↓ Top 5-10结果 → LLM
```

### 4. 数据飞轮机制

```
用户交互
    │
    ├── 显式反馈：点赞、评分、修正答案
    ├── 隐式反馈：停留时长、点击位置
    │
    ↓
数据收集层
    │
    ├── 反馈数据清洗
    ├── 多源数据整合（销售、客服、供应链）
    │
    ↓
评估分析层
    │
    ├── 模型效果评估（标准测试集+业务反馈）
    ├── 关键指标监控（性能、转化率、复购率）
    │
    ↓
优化迭代层
    │
    ├── 微调准入标准判断
    ├── 24h内微调或强化学习
    │
    ↓
模型更新 → 业务应用 → 新一轮用户交互
```
