## 九、核心术语表

### GraphRAG 核心术语

| 术语 | 定义 |
|------|------|
| **GraphRAG** | 图检索增强生成，通过知识图谱增强RAG的检索和生成，解决传统RAG上下文断裂和多跳推理不足 |
| **TextUnit** | GraphRAG索引的最小分析单元，即文本分块（chunk），默认200 token |
| **Entity Extraction** | 使用LLM从TextUnit中识别并提取实体（人名、地点、组织等）和实体间关系 |
| **Leiden算法** | 层次聚类社区检测算法，将知识图谱划分为层次化的社区结构，比Louvain更稳定 |
| **Community** | 图中紧密相关的实体和关系聚类，每个社区有LLM生成的摘要，用于全局搜索 |
| **Community Summary** | LLM为每个社区生成的自然语言摘要，捕捉社区内核心内容和关系 |
| **Global Search** | 基于社区摘要的Map-Reduce搜索，适合回答总结性/全局性问题（如"数据集主要主题"） |
| **Local Search** | 基于实体出发的精细化搜索，从查询实体扩展到关联节点和邻近社区，适合具体实体问题 |
| **DRIFT Search** | GraphRAG新增搜索模式，动态融合全局与局部搜索，先社区搜索再生成额外本地查询 |
| **Covariate（协变量）** | 与实体相关的统计数据或其他属性，如事件时间、影响范围 |
| **Claim** | 文本中表达的关键主张或论断，GraphRAG在抽取阶段同时提取 |

### GraphRAG 开源生态术语

| 术语 | 定义 |
|------|------|
| **MS GraphRAG** | 微软开源GraphRAG框架（31k Star），核心：Leiden社区检测+Map-Reduce全局搜索，适合摘要生成 |
| **LightRAG** | 港大开源轻量级GraphRAG（29k Star），双层检索+增量更新+多模态，适合快速部署和动态数据 |
| **KAG** | 蚂蚁开源知识增强生成框架（8k Star），OpenSPG+逻辑形式推理+DIKW知识表示，适合专业领域推理 |
| **HippoRAG** | OSU NLP组开源（3k Star），模仿海马体记忆机制，PPR概率扩散多跳推理，适合事实问答 |
| **LLMFriSPG** | KAG的面向LLM友好的知识表示框架，兼容强Schema专业知识和弱Schema开放信息 |

### LangGraph 高级特性术语

| 术语 | 定义 |
|------|------|
| **SubGraph** | LangGraph中的嵌套子图，将大workflow拆为模块化子组件，支持独立开发和复用 |
| **Command** | LangGraph 2025 API 新增类型，允许节点动态指定下一步路由（goto字段），替代条件边 |
| **Send** | LangGraph并行分发API，从单节点动态创建多个并行任务，实现Map-Reduce模式 |
| **Superstep** | LangGraph并行执行单元，同一superstep内的所有节点并发执行，全部完成后才进入下一步 |
| **Fan-Out** | 一个节点输出到多个后继节点，形成并行分叉 |
| **Fan-In** | 多个并行分支的结果汇聚到单个节点，需要reducer合并策略 |
| **Reducer** | State字段的自定义合并函数，用于合并并行分支的输出（如 operator.add 合并列表） |
| **State Transformation** | Parent Graph与SubGraph间状态结构不同时，用节点函数管理状态转换 |
