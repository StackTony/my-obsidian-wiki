
## vmcore 堆栈解析字段

### `bt` 命令输出堆栈字段解析

```
crash> bt
PID: 1234    TASK: ffff880123456000   CPU: 2    COMMAND: "nginx"
 #0 [ffff880123456000] machine_kexec at ffffffff8105c7bb
 #1 [ffff8801234567f8] crash_kexec at ffffffff8110a2d2
 #2 [ffff8801234568c0] oops_end at ffffffff81038d60
```

| 字段                    | 含义                                                          |
| --------------------- | ----------------------------------------------------------- |
| **PID**               | 进程ID，唯一标识一个进程                                               |
| **TASK**              | `task_struct` 结构体的地址，可通过 `struct task_struct <addr>` 查看详细信息 |
| **CPU**               | 崩溃时进程运行的CPU核心编号                                             |
| **COMMAND**           | 进程名称/命令名                                                    |
| **\#N**               | 栈帧编号，#0 是最内层（崩溃点），数字越大离崩溃点越远                                |
| **[addr]**            | 该栈帧的栈指针地址（RSP/RBP 或 SP/FP 位置）                               |
| **function + offset** | 函数名及其在函数内的偏移量                                               |

## x86_64 常见寄存器概览

### 通用寄存器

| 寄存器 | 全称 | 用途 |
|--------|------|------|
| **RAX** | Accumulator | 函数返回值、算术运算 |
| **RBX** | Base | 通用寄存器，callee-saved |
| **RCX** | Counter | 循环计数、第4个参数 |
| **RDX** | Data | 算术运算、第3个参数 |
| **RSI** | Source Index | 字符串操作源、第2个参数 |
| **RDI** | Destination Index | 字符串操作目标、**第1个参数** |
| **RBP** | Base Pointer | **栈帧基址**，用于定位局部变量 |
| **RSP** | Stack Pointer | **栈指针**，指向栈顶 |
| **R8-R15** | Extended | 通用寄存器，R9为第6个参数 |

### x86_64 函数调用约定 (System V AMD64 ABI)

| 参数位置 | 寄存器 |
|----------|--------|
| 第1个参数 | RDI |
| 第2个参数 | RSI |
| 第3个参数 | RDX |
| 第4个参数 | RCX |
| 第5个参数 | R8 |
| 第6个参数 | R9 |
| 更多参数 | 通过栈传递 |
| 返回值 | RAX |

### 指令指针与标志寄存器

| 寄存器 | 全称 | 用途 |
|--------|------|------|
| **RIP** | Instruction Pointer | **下一条要执行的指令地址**，崩溃分析关键 |
| **RFLAGS** | Flags Register | 状态标志位（零标志、进位标志等） |

### 段寄存器

| 寄存器 | 用途 |
|--------|------|
| **CS** | Code Segment - 代码段 |
| **DS** | Data Segment - 数据段 |
| **SS** | Stack Segment - 栈段 |
| **ES/FS/GS** | 附加段，内核常用 GS 存储每CPU变量 |

### x86_64 常见崩溃场景寄存器分析

```
RIP: 0010:[<ffffffff81234567>] do_page_fault+0x17/0x4f0
RSP: 0018:ffff8801234567e8  EFLAGS: 00010286
RAX: 0000000000000000 RBX: ffff880123456000 RCX: 0000000000000000
RDX: 0000000000000001 RSI: 0000000000000082 RDI: ffff8801234567e8
RBP: ffff880123456870 R08: 0000000000000000 R09: 0000000000000001
```

| 字段 | 分析要点 |
|------|----------|
| **RIP** | 崩溃发生的指令地址，用 `dis -s <addr>` 查看源码 |
| **RSP** | 栈指针，检查栈是否溢出或损坏 |
| **RDI/RSI/RDX** | 函数入参，结合源码分析参数是否合法 |
| **RBP** | 栈帧基址，用于回溯调用链 |

## ARM64 (AArch64) 常见寄存器概览

### 通用寄存器

| 寄存器 | 全称 | 用途 |
|--------|------|------|
| **X0-X7** | General Purpose | **参数寄存器**，X0 也是函数返回值 |
| **X8** | Indirect Result | 结构体返回值间接指针 |
| **X9-X15** | General Purpose | 临时寄存器，caller-saved |
| **X16-X17** | IP0/IP1 | 过程调用临时寄存器，用于PLT |
| **X18** | Platform Register | 平台保留，TLS或特殊用途 |
| **X19-X28** | General Purpose | callee-saved，函数内必须保存 |
| **X29** | Frame Pointer (FP) | **栈帧基址**，等价于 x86 的 RBP |
| **X30** | Link Register (LR) | **返回地址**，函数调用后返回地址 |
| **SP** | Stack Pointer | **栈指针**，等价于 x86 的 RSP |
| **PC** | Program Counter | **程序计数器**，等价于 x86 的 RIP |

### ARM64 函数调用约定 (AAPCS64)

| 参数位置 | 寄存器 |
|----------|--------|
| 第1个参数 | X0 |
| 第2个参数 | X1 |
| 第3个参数 | X2 |
| 第4个参数 | X3 |
| 第5个参数 | X4 |
| 第6个参数 | X5 |
| 第7个参数 | X6 |
| 第8个参数 | X7 |
| 更多参数 | 通过栈传递 |
| 返回值 | X0 |

### 特殊用途寄存器

| 寄存器 | 全称 | 用途 |
|--------|------|------|
| **PC** | Program Counter | 当前执行的指令地址 |
| **SP** | Stack Pointer | 栈指针 |
| **X29/FP** | Frame Pointer | 栈帧基址，用于回溯 |
| **X30/LR** | Link Register | 函数返回地址，`bl` 指令自动设置 |
| **PSTATE** | Processor State | 状态寄存器，类似 x86 的 RFLAGS |

### ARM64 异常级别 (Exception Level)

| EL | 全称 | 用途 |
|----|------|------|
| **EL0** | Exception Level 0 | 用户态应用程序 |
| **EL1** | Exception Level 1 | **内核态**，Linux内核运行级别 |
| **EL2** | Exception Level 2 | 虚拟化，Hypervisor |
| **EL3** | Exception Level 3 | Secure Monitor，ARM TrustZone |

```

```

| 字段 | 分析要点 |
|------|----------|
| **PC** | 崩溃发生的指令地址，用 `dis -s <addr>` 查看源码 |
| **LR (X30)** | 函数返回地址，用于回溯调用链 |
| **SP** | 栈指针，检查栈溢出或损坏 |
| **X0-X7** | 函数入参，结合源码分析参数合法性 |
| **FP (X29)** | 栈帧基址，用于栈回溯 |

### ARM64 栈回溯关键点

```
栈帧结构:
+------------------+ <- 高地址
|  局部变量        |
+------------------+
|  保存的 FP (X29) | <- 当前 FP 指向这里
+------------------+
|  保存的 LR (X30) |
+------------------+ <- 低地址 (SP)
```

栈回溯过程：`FP -> 保存的FP -> 保存的LR -> 上一级FP -> ...`

## 常见错误分析

### 空指针解引用

```
BUG: unable to handle kernel NULL pointer dereference at 0000000000000123
```

- 地址为 `0x123` → 结构体成员偏移 `0x123` 处，指针本身为 NULL
- 查看 `bt` 找到崩溃函数，检查入参

### 非法地址访问

```
BUG: unable to handle kernel paging request at ffffdeadbeef0000
```

- 地址看起来像"毒值"（如 `0xdeadbeef`）→ 可能是使用后释放（UAF）

### 栈溢出

```
stack segment: 0000 [#1] SMP
```

- SP 指向非法地址，检查是否有无限递归或局部变量过大
