## RAG 流水线关键术语图示

### 传统 RAG 流程
```
用户 Query
    ↓
【查询分析】
    ├── Query Transformation（查询改写）
    ├── HyDE（假设性文档嵌入）
    └── CRAG（纠错式RAG）
    ↓
【检索阶段】
    ├── Bi-Encoder → ANN（粗排）
    ├── BM25/SPLADE（关键词检索）
    └── Hybrid Search（混合检索）
    ↓
【融合阶段】
    └── RRF（倒数排名融合）
    ↓
【重排序】
    └── Cross-Encoder → Rerank（精排）
    ↓
【生成阶段】
    └── LLM → 答案 + 来源标注
    ↓
【评估指标】
    ├── Recall@K、Precision@K
    ├── NDCG、MRR
    └── Faithfulness、Answer Relevance
```

### GraphRAG 流程
```
原始文档 → TextUnits（文本单元分割）
    ↓
【知识图谱构建】
    ├── Entity/Relation/Claim 提取
    ├── Entity Resolution（实体对齐去重）
    └── Leiden Community Detection（社区检测）
    ↓
【分层摘要生成】
    ├── 全局摘要（顶层社区）
    ├── 社区摘要（中层）
    └── 局部摘要（底层）
    ↓
【查询阶段】
    ├── Global Search：匹配全局/社区摘要
    ├── Local Search：实体链接 → 子图提取
    └── 证据路径序列化 → LLM生成
```

### Agentic RAG 流程（Agent Loop）
```
用户 Query
    ↓
【Router Agent 判断查询类型】
    ├── 文档查询 → Vector RAG
    ├── 结构化查询 → Text-to-SQL RAG
    ├── 关系推理 → GraphRAG
    └── 简单问答 → 直接生成
    ↓
【Agent Loop 决策循环】
    ┌─────────────────────────────┐
    │  检索 → 评估检索结果        │
    │    ↓                        │
    │  结果不足？→ Query重写 → 再检索 │
    │    ↓                        │
    │  结果充足？→ Rerank → 生成    │
    │    ↓                        │
    │  Self-Correction 自纠错检查  │
    └─────────────────────────────┘
    ↓
【多Agent协作】
    ├── Planner Agent（规划拆解）
    ├── Retriever Agent（检索执行）
    ├── Validator Agent（验证准确性）
    └── Coordinator Agent（协调汇总）
```

## 一、核心架构概念
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|RAG|检索增强生成|先检索外部知识库上下文，再交给大模型生成答案|
|Grounding|事实锚定|限制大模型仅依托检索内容作答，减少幻觉|
|Hallucination|模型幻觉|大模型编造不存在事实、脱离知识库乱生成内容|
|Context Window|上下文窗口|大模型单次可容纳处理的最大 Token 长度|
|Token|词元|大模型文本最小计算单位，中英文拆分规则不同|
|VRAG|视觉检索增强生成|图文多模态RAG，图文联合检索问答|
|MRAG|多模态检索增强生成|融合文本、图片、音频、视频全模态RAG|
|CRAG|纠错检索增强生成|带校验纠错机制，过滤错误检索内容|
|Self-RAG|自检索增强生成|模型自主判断是否需要检索、自主筛选检索内容|
|Agentic RAG|智能体RAG|结合AI智能体，多轮规划检索+工具调用完成复杂问答|
|Naive RAG|基础原生RAG|最简分块+向量检索+prompt拼接，无优化流程|
|Advanced RAG|进阶优化RAG|融合重排、查询改写、多级索引等优化方案|

## 二、向量与嵌入技术
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|Embedding|文本嵌入|将文本转为数值向量，实现语义量化比对|
|Dense Vector|稠密向量|全维度浮点向量，主打语义相似度匹配|
|Sparse Vector|稀疏向量|仅关键词有权重，主打关键词精准匹配|
|Bi-Encoder|双编码器|分别独立编码问句、文档，用于快速粗召回|
|Cross-Encoder|交叉编码器|拼接问句+文档联合编码，用于高精度重排|
|SPLADE|稀疏词嵌入模型|生成学习型稀疏向量，兼顾语义与关键词|
|Sentence-BERT|句向量模型|专门优化短句语义嵌入的轻量编码模型|
|Multi-Q Embedding|多查询嵌入|生成多条相似问句向量，提升召回覆盖面|
|Matryoshka Embedding|套娃嵌入|支持任意维度裁剪向量，兼顾速度与精度|
|Instruction Embedding|指令型嵌入|带任务指令训练，适配检索专属语义表达|

## 三、检索算法体系
### 1. 向量相似度检索
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|KNN|精确最近邻搜索|全局暴力匹配，结果最准、速度慢|
|ANN|近似最近邻搜索|牺牲极小精度，大幅提速向量相似度检索|
|HNSW|层次导航小世界|工业界主流ANN向量索引算法，速度快精度高|
|FAISS|向量检索库|Meta开源高性能向量检索工具库|
|IVF|倒排文件索引|聚类分桶向量索引，海量数据提速检索|
|PQ|乘积量化|向量压缩算法，降低内存占用、加速检索|

### 2. 传统关键词检索
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|TF-IDF|词频逆文档频率|基础关键词权重计算算法|
|BM25|最佳匹配25|工业级经典稀疏关键词检索算法|
|倒排索引|词条反向索引|建立词语到所属文档的映射，加速关键词查找|
|ELK分词检索|开源日志检索|基于分词+倒排的全文检索方案|

### 3. 检索融合&优化
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|RRF|倒数排名融合|融合多路检索结果，平衡语义与关键词召回|
|Rerank|结果重排序|对粗召回候选集做相关性二次精准打分排序|
|Top-K|前K召回|检索时优先返回相关性最高的 K 条文档片段|
|Filter|检索过滤|基于时间、标签、目录、权限等条件筛选结果|
|Query Expansion|查询扩展|对用户问题做同义改写、补全，提升召回覆盖率|
|HyDE|假设文档检索|让模型先脑补答案，再用脑补文本做语义检索|
|CoT Retrieval|思维链检索|借助思维链拆解复杂问题，分步精准检索|
|Sub-Query|子问题拆分|复杂问句拆分为多个简单子问题分别检索|
|Score Fusion|分数融合|合并向量分数、关键词分数统一排序|

## 四、文本预处理&分块技术
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|Chunking|文本分块|长文档切分为固定/语义片段，适配入库与检索|
|Recursive Splitting|递归字符分块|按层级分隔符逐级切割，通用稳定分块方案|
|Semantic Chunking|语义分块|依靠向量相似度识别语义边界，自然切分文本|
|Overlap|分块重叠|相邻分片预留重叠内容，防止上下文信息断裂|
|jieba|结巴分词|主流轻量中文分词工具，适配中文检索预处理|
|StopWords|停用词|无实际语义虚词，检索/分词阶段直接过滤剔除|
|Markdown Splitter|富文本分块|适配标题、列表、代码块结构化文档切割|
|Code Splitting|代码专属分块|按函数、类、语句块拆分程序代码文档|
|Cleaning|文本清洗|剔除乱码、空格、特殊符号、水印等无效内容|
|Normalization|文本归一化|繁简转换、大小写统一、同义词标准化处理|

## 五、知识库索引架构
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|Vector DB|向量数据库|专门存储、索引、检索海量文本向量的数据库|
|Hierarchical Index|分层索引|构建文档-章节-段落多级知识库索引结构|
|Parent Document Retrival|父文档检索|小块召回、大块拼接上下文作答，兼顾精度与完整度|
|Child Chunk|子分块|最小检索单元，用于精准定位信息|
|Graph RAG|图谱检索增强|结合知识图谱，实现实体、关系级精准检索|
|Knowledge Graph|知识图谱|存储实体、属性、关联关系的结构化知识库|
|Namespace|向量库命名空间|向量库数据隔离分区，区分不同业务知识库|
|Snapshot|知识库快照|备份知识库向量与索引，用于回滚与迁移|
|Incremental Update|增量更新|仅新增修改文档入库，无需全量重建索引|

## 六、RAG 流程链路术语
|术语|中文释义|简明解释|
| ---- | ---- | ---- |
|Data Ingestion|数据接入|原始文档导入、解析、入库全流程|
|Pipeline|检索流水线|预处理-分块-嵌入-入库-检索-重排标准化流程|
|Prompt Template|提示词模板|固定格式指令，规范模型使用检索上下文作答|
|Context Injection|上下文注入|将检索到的片段拼接入大模型输入对话|
|Response Synthesis|答案合成|整合多条检索内容，梳理逻辑生成通顺回答|
|Citation|引用溯源|答案标注对应原文出处、段落位置|
|Retrieval Evaluation|检索评测|用准确率、召回率评估检索效果|
|MRR|平均倒数排名|检索结果排序效果核心评估指标|
|Precision|精准率|召回内容里有效相关内容占比|
|Recall|召回率|全部相关内容被成功找出的比例|
